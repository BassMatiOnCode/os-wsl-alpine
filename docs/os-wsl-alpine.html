Alpine Linux on WSL

Alpine is a very light-weight version of Linux.

Installation

Firstly, make sure that that the Windows Subsystem for Linux is activated: Open Control Panel\Programs\Programs and Features, click Turn Windows features on or off, tick the component Windows Subsystem for Linux and click OK. Then open a command prompt and execute

wsl --update

This ensures that WSL2 is available. Then go to the Microsoft Store and install Alpine Linux by agova338 (the official maintainer of the MS Store package. When done, start the installation. You are asked to create a new user and password. This user will be added to the wheel group and therefore be able to run doas (Alpine's sudo replacement.

If you get error messages like "cannot translate path", don't let that confuse you. Alpine does not mount removable disks automatically, and these folders might be referenced from your Windows path environnment variable. Hence the errors. We'll cover that later.

Finally, you should end up at the command prompt, similar to this:

computername:/path/to/folder#

The # at the end of the prompt string indicates that you are root. A $ appears if you are logged in as normal user.

If you must open a root console anytime later, run the following command in Windows:

wsl -d Alpine -u root

This might be necessary to reset a user password, for instance.

Now that we are still root, install doas, Alpine's sudo replacment. Exedute the following statements:

apk add doas
echo permit persist :wheel > /etc/doas.d/20-wheel.conf

This creates a doas configuration file that allows members of the wheel group to run doas and execute command with admin permission.

Now terminate the console session, execute

exit

Back in Windows, open a console window and run the following command:

wsl -d Alpine

This runs the Alpine distribution and opens a terminal for the default user:

computername:~$

The tilde indicates your home directory. The $ indicates that you are NOT root. Now let's try admin elevation:

doas apk update

This command invokes the Alpine Package Keeper and runs the update command. This requires root access, therefore the doas prefix. You are asked to enter your password. At this point, after a fresh installation, my user password was never accepted. It seems that the doas installation resets or deletes the user password. Therefore, reset the password and try again:

$ exit
> wsl -d Alpine -u root
# passwd username
# exit
> wsl -d Alpine
$ doas apk update

NOTE that you have to type the password blindly, for security reasons. Finally, doas and apk update should run successfully.

Mounting1 Removable Disks

In contrast to fixed disks, removable disks are not mounted automatically.

There are several ways to achieve that, some work, others don't. The fstab method didn't work out for me.

The first unpleasant issue that hit me was Alpine trying to integrate Windows path environment variables. Some entries on my system point to a removable disk, and SD card, with drive letter D:. At (Linux) system startup, removable drives have not yet been mounted. As a result, several error messages are generated, saying that a specific path could not be translated. To suppress these messages, I added an entry to /etc/wsl.conf:

[interop]
appendWindowsPath = false

This silenced the system startup beautifully.

The next thing that hit me was mounting removable drives into the Linux file system. Entries in /etc/fstab also produced errors at system startup, so I removed them again. Additionally, I added these lines to /etc/wsl.conf:

[automount]
enabled = true
mountFsTab = false
root = /mnt/
options = "metadata,uid=1000,gid=1000,umask=22"

[network]
generateResolvConf = true

Several attempts to use WSL features to get removable disks mounted automatically all failed, so I gave up and resortet to a reliable scripted method: Shell startup scripts.

I created the file /etc/profile.d/mount-sd.sh. Files in this directory that end on .sh are executed by the shell startup script. Inside is the following script:

#!/bin/sh
# Mount the SD card as drive D:
if ! mountpoint -q /mnt/d; then
    [ ! -d /mnt/d ] && sudo mkdir -p /mnt/d
    sudo mount -t drvfs D: /mnt/d -o metadata,uid=1000,gid=1000,umask=22 > /dev/null 2>&1
fi

# Mount Sandisk USB stick as F:
if ! mountpoint -q /mnt/f; then
    [ ! -d /mnt/f ] && sudo mkdir -p /mnt/f
    sudo mount -t drvfs F: /mnt/f -o metadata,uid=1000,gid=1000,umask=22 > /dev/null 2>&1
fi

The script creates the mount points (/mnt/d and mnt/f) and mounts the corresponding Windows drives, the SD card D: and a USB thumb drive F:.
